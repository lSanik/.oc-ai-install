# Commands — Команди AI

## ІНСТРУКЦІЯ ДЛЯ AI

**Єдине джерело** тексту для `/scope`, `/review`, `/update_ai_doc`, нагадувача та блоку про середовище. Схема `scheme-cursorrules.md` **не дублює** ці розділи — при генерації `.cursor/rules/commands.mdc` копіюй вміст звідси **до розділу «Розбиття для Claude Code»** (той розділ лише для гілки Claude). Файли `.claude/commands/*.md` — з цього ж файлу, розбиття див. в кінці.

**Головний файл правил** залежить від інструмента:
- **Claude Code:** `.claude/CLAUDE.md`
- **Cursor:** `.cursor/rules/main.mdc`

У кроках нижче під «головний файл правил» мається на увазі відповідний шлях у **цьому** проєкті.

## /scope — Планування задачі

**Коли використовувати:** перед початком нової задачі.

**Що робить:**
1. Якщо є незакриті задачі — повідомляє про них. Якщо користувач **починає нову тему** без явного «готово» / «є правки» — **не** закривай стару мовчки: **попередь**, перелічи відкриті задачі і **запитай**, чи відкривати нову поверх них чи спочатку закрити стару (узгоджено з `scheme-ai-task.md` / записом у `ai-task.md`).
2. Читає `ai-map.md` і **головний файл правил**
3. Відповідає планом:

```
УВАГА: є незакриті задачі: [список] (якщо є)

SCOPE: [назва задачі]

Торкнемось файлів:
- [файл] — [що змінюємо]

НЕ чіпаємо:
- [файл] — [чому]

Warning Zone на шляху:
- [файл] — [попередження]

Рівень змін: Критичні / М'які
Ризики: [список]

Відкрити задачу?
```

## /review — Перевірка після задачі

**Коли використовувати:** після завершення роботи, перед закриттям задачі.

**Що робить:**
1. Читає `ai-task.md` — бере список змінених файлів з лог змін
2. Для кожного перевіряє:
   - Логіка (очевидні помилки)
   - Безпека (SQL-ін'єкції, XSS, відкриті дані)
   - Залишки дебагу (var_dump, echo, print_r, dd(), die()) — **виняток:** у **`migration.php` у корені** OpenCart перший виконуваний рядок після `<?php` з навмисним `die(0);` / `die();` не вважати дебагом
   - Відповідність **головному файлу правил** і `code-style.md`
3. Виводить звіт:

```
REVIEW: [назва задачі]

[файл] — ok
[файл] — знайдено: [опис]
[файл] — критично: [опис]

Залишки дебагу: [список або "немає"]
Рекомендації: [або "все чисто"]
```

## /update_ai_doc — Оновлення документації

**Коли викликається:** автоматично при закритті задачі.
**Можна викликати вручну** якщо потрібно оновити документацію без закриття задачі.

**Алгоритм:**

1. Читає поточну задачу з `ai-task.md`
2. Визначає які AI-файли потребують оновлення:
   - Якщо змінювалась БД → оновити `db_mapping.md`
   - Якщо новий модуль → оновити `ai-map.md`
   - Якщо змінились домовленості про стиль, DRY, події або межі catalog/admin → оновити `code-style.md`
   - Якщо нова Warning Zone → оновити `ai-map.md`, **головний файл правил**; у Cursor також `.cursor/rules/warning-zone.mdc` (список і опис збігаються з `ai-map.md`)
   - Завжди → додати запис в `ai-changelog.md`
   - Якщо архітектурне рішення → додати ADR в `ai-decisions.md`
3. Для кожного зміненого AI-файлу — актуалізує вміст за потреби (без нумерації версій файлів).
4. Виводить підсумок:

```
/update_ai_doc завершено

Оновлено:
- [файл] — [коротко що змінено]

Додано в ai-changelog.md: [коротко]
```

## Нагадувач (не команда — поведінка AI)

Поки є відкриті задачі — AI додає в кінець **кожної** відповіді:

**Одна задача:**
```
---
Задача відкрита: [назва] [критичні/м'які]
Скажи "готово" або "є правки: [опис]" щоб закрити.
```

**Кілька задач:**
```
---
Відкриті задачі (N): [КРИТ] [назва] [М'ЯК] [назва]
"готово [назва]" або "готово всі"
```

## Важливо про середовище

`CAN_RUN_COMMANDS` стосується CLI-інструментів у середовищі проєкту: `php`, `php -l`, `php -v`, `composer`, `npm`, `git` та подібних. (Наприклад, `php artisan` — Laravel, але принцип той самий.)

Якщо `CAN_RUN_COMMANDS = no` — не виконувати такі команди автоматично через Bash.
Виводити команду текстом і просити виконати вручну:

```
Виконай будь ласка:
$ [команда]

І поділись результатом.
```

## Розбиття для Claude Code (`.claude/commands/`)

З одного цього файлу згенеруй **три** файли (кожен з заголовком `#` за назвою команди або першим рядком змісту):

| Файл | Що копіювати з цього документа |
|------|--------------------------------|
| `commands/scope.md` | Розділ `## /scope` (повністю) |
| `commands/review.md` | Розділ `## /review` (повністю) |
| `commands/update_ai_doc.md` | Розділ `## /update_ai_doc` (повністю) |

У **кінець кожного** з трьох файлів додай розділи `## Нагадувач` та `## Важливо про середовище` з цього ж файлу (той самий текст, без змін).

У кроках читання **головного файлу правил** у згенерованих файлах для Claude вказуй шлях `.claude/CLAUDE.md`.
