# .ai-oc-install — AI Onboarding Wizard for OpenCart

A setup wizard that configures **Claude Code** or **Cursor** to work intelligently with your OpenCart / ocStore project. Run it once — the AI gets a full picture of your project, follows your code style, and never touches sensitive files.

---

## What it does

The wizard asks you a series of questions about your project and generates a ready-to-use AI configuration:

| Generated file | Purpose |
|---|---|
| `.claude/CLAUDE.md` or `.cursor/rules/main.mdc` | Entry point: tells the AI who it is and what rules to follow |
| `.claude/project.md` / `.cursor/project.md` | Project data: platform version, theme, languages, environment, constraints |
| `.claude/ai-map.md` / `.cursor/ai-map.md` | Map of custom modules and deviations from OpenCart standard structure |
| `.claude/code-style.md` / `.cursor/code-style.md` | Code style rules for PHP, JS, CSS, templates |
| `.claude/settings.json` | Claude Code permissions: deny list for destructive commands |
| `.ai-oc-install/map/db_mapping.md` | DDL / description of custom and modified database tables |

Sensitive files (`config.php`, `.env`, `admin/config.php`, `system/config/`, etc.) are blocklisted automatically — the AI will refuse to read or modify them.

Say **review / ревью / перевір задачу** after any task and the AI runs a 12-point OpenCart checklist: PHP syntax, debug cleanup, architecture rules, DB escaping, cache, security, performance, and more.

---

## Who it is for

- OpenCart or ocStore developers who use **Claude Code** (CLI) or **Cursor** as their AI assistant.
- Teams that want the AI to understand project-specific rules, custom folder names, theme structure, and database schema from day one.

---

## How to use

1. Clone or copy this repository into the root of your OpenCart project as `.ai-oc-install/`.
2. Open Claude Code or Cursor in the project root.
3. Tell the AI:
   ```
   Read .ai-oc-install/ai-oc-install.md and start the installer.
   ```
4. Answer the questions block by block (platform version, PHP, environment, theme, languages, custom folder, DB tables, warning files).
5. The wizard generates all configuration files and runs a post-generation check.

That's it. On the next session the AI loads `CLAUDE.md` / `main.mdc` automatically and is ready to work.

---

## Reinstall

If the rules in `.ai-oc-install/` are updated and you want to regenerate the AI config without losing your project data:

```
Read .ai-oc-install/ai-oc-install.md and reinstall.
```

The wizard reads the existing `project.md`, skips all questions, deletes only the regeneratable files, and rewrites them from the current schemes. Your `project.md`, `ai-map.md`, and `db_mapping.md` are never touched.

## Global instructions (optional, recommended)

The repository includes two ready-made files with baseline AI behavior rules that work across **all your projects**, not just OpenCart ones.

| File | For |
|---|---|
| `claude-global-instructions-light.txt` | Claude Code |
| `cursor-global-instructions-light.txt` | Cursor |

Both files cover the same ground:

- **Language** — respond in English or Ukrainian (Ukrainian by default).
- **Project rules** — always prefer project-level config (`CLAUDE.md` / `main.mdc`) when present; warn once if it is missing.
- **Code quality and scope** — no bulk fixes, no out-of-scope refactoring, comment only non-obvious things, prefer early returns, avoid magic numbers.
- **Closures** — named methods by default; closures only when short and local; extract to a method if they grow past ~5 lines.
- **Terminal** — never run `php` / `php artisan` unless the user explicitly asks.
- **Architecture** — follow existing patterns; suggest before introducing new ones.
- **Task workflow** — assess complexity, plan for medium/complex tasks, confirm before large edits, then execute with tight scope.

### How to apply

**Claude Code** — paste the content of `claude-global-instructions-light.txt` into your global `~/.claude/CLAUDE.md` (or use the Claude Code settings UI: *Custom instructions*).

**Cursor** — paste the content of `cursor-global-instructions-light.txt` into *Cursor Settings → Rules for AI → User Rules* (the global rules field, not a project `.mdc` file).

These global rules are intentionally minimal ("light"). They establish sensible defaults and then defer to per-project config generated by this installer.

## Repository structure

```
.ai-oc-install/
├── ai-oc-install.md          # Main installer instructions (entry point for the AI)
├── global/
│   ├── blocklist.md          # Red Zone: files the AI must never read or modify
│   ├── git.md                # Git / .gitignore guidance
│   ├── warning-zone.md       # Template for files that need extra care
│   └── review.md             # Review checklist: triggered by "review / ревью / перевір задачу"
├── opencart/
│   ├── main.md               # Core OpenCart architecture overview
│   ├── controller.md         # MVC: controllers
│   ├── model.md              # MVC: models (includes migration.php rules)
│   ├── view.md               # MVC: templates
│   ├── language.md           # Language files
│   ├── system-library.md     # system/library/
│   ├── admin.md              # Admin panel specifics
│   ├── catalog.md            # Storefront specifics
│   ├── php.md                # PHP coding rules
│   ├── js.md                 # JavaScript rules
│   ├── css.md                # CSS rules
│   └── mysql.md              # Database / query rules
├── schemes/
│   ├── scheme-claude-md.md   # Template for CLAUDE.md
│   ├── scheme-cursorrules.md # Template for Cursor rules
│   ├── scheme-project.md     # Template for project.md
│   ├── scheme-ai-map.md      # Template for ai-map.md
│   ├── scheme-db-mapping.md  # Template for db_mapping.md
│   ├── scheme-code-style.md  # Template for code-style.md
│   └── scheme-settings.md    # Template for Claude settings.json
├── map/                              # Created during install (shared by Claude and Cursor)
│   ├── db_mapping.md                 # Your custom / modified table DDL and notes
│   ├── db_map.php                    # Optional: generated index of table names
│   └── db_tables/                    # Optional: one .php file per table with full CREATE TABLE
├── getMap.php                        # Optional helper: generates db_map.php and db_tables/
├── claude-global-instructions-light.txt  # Global baseline rules for Claude Code (all projects)
└── cursor-global-instructions-light.txt  # Global baseline rules for Cursor (all projects)
```
